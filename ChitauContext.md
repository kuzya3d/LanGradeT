Проект был расширен из базового Laravel-сайта для изучения английского в более полноценную учебную платформу:

- личный словарь;
- тематические подборки слов;
- начальный тест уровня;
- несколько тренировочных тестов;
- XP, рейтинг и достижения;
- избранные уроки и наборы;
- внутренний чат с друзьями;
- локальный AI tutor по базе сайта;
- современный адаптивный фронтенд в зеленом стиле LanGrade.

## Сидеры

Сидеры были сильно расширены. Важно: порядок сидеров имеет значение, потому что поздние сидеры исправляют и дополняют ранние.

Основные сидеры:

- `CollectionsTableSeeder` - базовые коллекции.
- `WordsTableSeeder` - базовые слова.
- `CollectionWordSeeder` - начальная привязка слов к коллекциям.
- `LearningContentSeeder` - типы тестов, первые уроки, предложения, достижения.
- `ExpandedLearningSeeder` - дополнительные темы, слова, предложения и уроки.
- `RichLessonsSeeder` - расширенный HTML-контент уроков.
- `ComprehensiveVocabularySeeder` - большой набор слов по темам.
- `SourceInspiredContentSeeder` - дополнительные слова/предложения по мотивам частотной лексики и Present Simple.
- `CollectionRepairAndBulkVocabularySeeder` - ремонт логики коллекций и пополнение словаря.
- `GrammarConceptsSeeder` - урок по базовым грамматическим понятиям.
- `CleanVocabularySeeder` - финальная чистка плохих искусственных словоформ.

Важно: ранее база искусственно добивалась до 1000+ через word-family формы, но это оказалось плохим UX. Поэтому `CleanVocabularySeeder` удаляет записи вроде:

- `accepts`;
- `bank (мн. число)`;
- `образ действия: ...`;
- `состояние: ...`;
- `verb form`;
- `noun plural`;
- `word family`.

В базе должны оставаться нормальные учебные слова формата `english` + `russian`.

## Тесты

Старые и новые тесты приведены к общей системе результатов:

- сохраняется попытка в `test_attempts`;
- сохраняются ответы в `test_answers`;
- начисляется или снимается XP;
- показывается подробный разбор: вопрос, ответ пользователя, правильный ответ, верно/неверно.

Тесты:

- `/tests/multiple-choice` - выбор перевода.
- `/tests/gap-fill` - пропущенное слово.
- `/tests/sentence-builder` - сборка предложения.
- `/tests/phonetics` - слово по транскрипции.
- `/tests/word-sprint` - быстрый true/false по паре слово-перевод.
- `/tests/compile-word` - сборка слова из букв.
- `/tests/translation` - ввод перевода.
- `/tests/flashcards` - карточки без XP.

Особенности:

- `multiple-choice` имеет таймер 60 секунд. При окончании времени отправляются только уже пройденные/отвеченные вопросы.
- `/tests/multiple-choice?source=dictionary` считается закреплением личного словаря: попытка и разбор сохраняются, но XP не начисляется и не снимается, как у flashcards.
- `sentence-builder` имеет два режима:
  - печатать ответ;
  - собирать предложение плитками.
- За печатный режим sentence-builder XP выше.
- `compile-word` добавляет лишние случайные буквы для усложнения.
- `flashcards` временно не использует фотографии, вместо них буквенный визуал.
- Enter в формах тестов предотвращает случайный submit.

Проверка ответов стала мягче:

- игнорируются регистр и знаки;
- допускаются мелкие опечатки через Levenshtein;
- но логика времен/грамматики пока базовая, без полноценного NLP.

XP:

- хороший результат начисляет XP;
- результат ниже 55% снимает немного XP;
- результат ниже 35% снимает больше XP;
- XP пользователя не уходит ниже нуля.

## AI tutor

`/ai-tutor` - локальный помощник без внешнего API.

Он умеет:

- искать слово в базе по английскому или русскому;
- искать неточно, с учетом небольших ошибок;
- возвращать перевод, транскрипцию, часть речи, пример;

Это не настоящий LLM, а rule-based помощник по базе сайта.

## Локальная транскрипция для личного словаря

Добавлена генерация IPA-транскрипции для слов, которые пользователь вручную добавляет в личный словарь через `/dictionary`.

Состав:

- `scripts/g2p_transcribe.py` - локальный Python-скрипт на `g2p-en`, превращает ARPABET из модели в IPA в квадратных скобках, например `hello` -> `[həˈloʊ]`.
- `app/Services/PhoneticTranscriber.php` - PHP-обертка для вызова скрипта из Laravel.
- `UserDictionaryController::store()` теперь при создании нового `Word` заполняет `transcription`; если слово уже было в базе без транскрипции, поле дозаполняется.
- `app/Console/Commands/FillMissingWordTranscriptions.php` добавляет команду `words:fill-transcriptions`, есть флаг `--only-user` для уже привязанных к пользователям слов.

Локальные зависимости были установлены в пользовательский Python:

```powershell

python -m nltk.downloader cmudict averaged_perceptron_tagger averaged_perceptron_tagger_eng
```

В `.env` добавлен путь:

```dotenv

```

Если Python переедет, нужно поменять `G2P_PYTHON`. После установки зависимостей генерация работает без интернета.

## GigaChat AI tutor

`/ai-tutor` переключен с прежней Google-интеграции на GigaChat API.

Состав:

- `app/Services/GigaChatTutorService.php` - получает OAuth access token через `POST /api/v2/oauth`, кеширует его по `expires_at` с запасом, при `401` сбрасывает кеш токена и повторяет запрос один раз, затем отправляет чат в `POST /api/v1/chat/completions`.
- `AiTutorController::ask()` не ходит во внешний API синхронно. Он записывает вопрос, создает pending-сообщение `Думаю...` и запускает отдельный процесс `ai:tutor-answer`, чтобы `php artisan serve` не зависал на внешнем HTTP-запросе.
- `AiTutorController::message()` отдает JSON-статус конкретного сообщения для frontend polling.
- `app/Console/Commands/AnswerAiTutorMessage.php` отвечает на pending-сообщение через GigaChat; при ошибке API/интернета/лимита сохраняет fallback с `meta.engine = local-fallback`.
- `app/Services/BackgroundCommandRunner.php` запускает artisan-команду в фоне на Windows/OSPanel.
- `resources/views/ai/index.blade.php` обновлен: отправка через `fetch`, без перезагрузки страницы; pending отображается как тихий текст `Думаю...`, затем заменяется готовым ответом через polling `/ai-tutor/messages/{message}`.
- `config/services.php` содержит настройки `services.gigachat`.

`.env`:

```dotenv
GIGACHAT_AUTHORIZATION_KEY=...
GIGACHAT_OAUTH_URL=https://ngw.devices.sberbank.ru:9443
GIGACHAT_API_URL=https://gigachat.devices.sberbank.ru
GIGACHAT_SCOPE=GIGACHAT_API_PERS
GIGACHAT_MODEL=GigaChat
GIGACHAT_TIMEOUT=12
GIGACHAT_CONNECT_TIMEOUT=5
GIGACHAT_VERIFY_SSL=false
GIGACHAT_PROXY=
```

`GIGACHAT_VERIFY_SSL=false` оставлен для локальной Windows/OSPanel-среды, потому что GigaChat часто требует корректные российские TLS-сертификаты. Если сертификаты установлены, можно вернуть `true`.

## Initial test, XP levels, streak, bio

`/initial-test` переработан в полноценный placement test:

- секции: слова, словосочетания, грамматика, понимание предложений;
- больше нет самооценки чекбоксами "знаю слово";
- добавлено больше простых A1/A2 слов, фраз и предложений, чтобы тест лучше отличал минимальный уровень от начального;
- ответы проверяются на сервере по структуре `InitialTestController::sections()`;
- уровень выше A0 дается только при достаточном общем проценте и минимуме по ключевым секциям;
- итоговый уровень не понижается ниже уже достигнутого пользователем уровня.
- на прохождение дается 5 минут; если таймер заканчивается, форма отправляется с `timed_out=1`, а placement-результат принудительно становится A0.

Пороги XP для автоматического повышения `users.english_level`:

- A1: 5000 XP;
- A2: 10000 XP;
- B1: 20000 XP;
- B2: 35000 XP;
- C1: 50000 XP.

Логика находится в `App\Models\User::levelFromXp()` и `promoteEnglishLevelFromXp()`. Она вызывается после начисления XP в `RecordsPractice` и при заходе пользователя через middleware `UpdateUserStreak`.
